# TUI Extension for Weave

## Overview
Add a fully-featured terminal UI to weave as an extension module using Bubble Tea. The TUI subscribes to the existing event bus for rendering (streaming messages, tool output, diffs) and publishes user input back to the bus. The `-p` flag retains its current behavior (print mode, no TUI). When `-p` is not set, the TUI extension is included in the build and provides an interactive chat experience.

A new `sdk.UI` interface enables cross-extension integration â€” extensions can show popups, set status bar text, register slash commands, and register custom keybindings.

## Context
- **Framework**: Bubble Tea (charmbracelet) with glamour (markdown), lipgloss (styling), bubbles (widgets)
- **Architecture**: TUI as an extension (`extensions/ui/tui/`), self-registering via `sdk.RegisterExtension("tui", ...)`
- **Key pattern**: Bridge goroutine translates between bus events (`<-chan sdk.Event`) and Bubble Tea messages (`tea.Msg`)
- **Selected features**: streaming+markdown chat, tool output rendering, thinking blocks+diffs, full editor, context bar, token stats, loading spinner, slash commands, session manager, model selector
- **Cross-extension**: `sdk.UI` interface with registry, `NoopUI` stub for print mode

## Technical Details

### New SDK types
```go
// sdk/ui.go
type UI interface {
    Select(title string, items []string) (int, error)
    Confirm(message string) (bool, error)
    Input(prompt string) (string, error)
    SetStatus(key, text string)
    Notify(message string)
    RegisterCommand(name string, handler func(args string) error)
    RegisterRenderer(toolName string, renderer ToolRenderer)
    RegisterKeybinding(kb Keybinding)
}

type Keybinding struct {
    Name        string
    Keys        []string
    Description string
    Handler     func() error
}

type ToolRenderer interface {
    Render(content string, width int) string
}
```

### TUI event flow
```
User types in editor â†’ editor submits â†’ bus.Publish("agent.prompt", text)
                                            â†“
Agent-loop receives prompt â†’ streams response â†’ bus.Publish("agent.message_update", delta)
                                            â†“
Bridge goroutine reads bus event â†’ program.Send(MessageUpdateMsg{Content: delta})
                                            â†“
Bubble Tea Update() appends delta to assistant message component
                                            â†“
View() renders full chat + updated message
```
