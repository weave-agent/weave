# Guardian Policy Overlay SDK API

## Overview
- Add SDK-level Guardian policy overlay event contracts so trusted extensions can apply temporary policy layers without creating user-visible Guardian profiles.
- Policy overlays are session-runtime controls that can tighten or loosen the active Guardian profile and can explicitly opt into hard-block override semantics.
- This repository owns the shared SDK event topics, payload types, snapshot shape, JSON compatibility, and generated mocks/docs needed by extension repos.
- Plan mode implementation is out of scope; this plan only adds the reusable overlay API surface.

## Context (from discovery)
- Files/components involved:
  - `sdk/guardian.go` defines Guardian event topics, Guardian payload structs, `GuardianSnapshot`, and `Guardian` interface.
  - `sdk/guardian_test.go` verifies event topics and JSON round-trips for Guardian payloads.
  - `sdk/guardian_mock_test.go` is generated by moq when interfaces change.
  - `CLAUDE.md` documents Guardian event topics and architecture rules.
- Related patterns found:
  - Existing Guardian events use SDK constants with typed payload structs.
  - Extensions communicate through bus events, not direct imports.
  - Snapshot additions are backward-compatible for Go callers that use named fields and JSON clients that ignore unknown fields.
- Dependencies identified:
  - `weave-guardian` will consume the new SDK topics and snapshot field.
  - `weave-tui-guardian` will consume `GuardianSnapshot.Overlays` for display.

## Technical Details
- Proposed topics:
  - `guardian.policy.overlay.push`
  - `guardian.policy.overlay.pop`
- Proposed payloads:
  - `GuardianPolicyOverlay` for adding/replacing an active runtime overlay
  - `GuardianPolicyOverlayPop` for removing an active runtime overlay by ID
- Proposed snapshot change:
  - `GuardianSnapshot.Overlays []GuardianPolicyOverlay`
- Overlay semantics are implemented in `weave-guardian`, not in this repository.
